# Rewrite Patterns

## Pattern 1: Generic Thesis -> Owned Tension

### Too AI-ish
技术人做 B 端，最重要的是建立信任并理解客户需求。

### Better
我后来才明白，技术人做 B 端，最难的不是把东西做出来，而是在客户还没看到结果之前，先让他愿意相信你。

## Pattern 2: Framework Voice -> Retelling Voice

### Too AI-ish
这件事的核心在于，我完成了从技术视角到商业视角的转变。

### Better
以前我脑子里装的全是功能、流程和实现细节，真到和客户谈单的时候才发现，对方在意的根本不是这些。

## Pattern 3: Over-Chopped Sentences -> Natural Flow

### Too AI-ish
我犹豫过。  
我担心过。  
我甚至想过先不做。  
因为这很冒险。

### Better
坦白讲，我一开始也犹豫过，因为在合同没签之前就先做东西，听起来多少有点冒险。

## Pattern 4: Abstract Summary -> Concrete Stakes

### Too AI-ish
客户真正关心的是效率和结果。

### Better
客户真正在意的，不是你用了什么模型，而是这件事能不能帮他少用几个人、少花多少时间，以及最后能不能顺利上线。

## Pattern 5: Fake Emotion -> Earned Emotion

### Too AI-ish
我非常激动，也非常感慨，这次经历让我收获很多。

### Better
钱打进来的那一刻，我第一个感受不是兴奋，而是终于松了一口气：原来这条路，真的能走通。
