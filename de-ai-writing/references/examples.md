# Rewrite Patterns

## Pattern 1: Generic Thesis -> Owned Tension

### Too AI-ish
技术人做 B 端，最重要的是建立信任并理解客户需求。

### Better
我后来才明白，技术人做 B 端，最难的不是把东西做出来，而是在客户还没看到结果之前，先让他愿意相信你。

## Pattern 2: Framework Voice -> Retelling Voice

### Too AI-ish
这件事的核心在于，我完成了从技术视角到商业视角的转变。

### Better
以前我脑子里装的全是功能、流程和实现细节，真到和客户谈单的时候才发现，对方在意的根本不是这些。

## Pattern 3: Over-Chopped Sentences -> Natural Flow

### Too AI-ish
我犹豫过。  
我担心过。  
我甚至想过先不做。  
因为这很冒险。

### Better
坦白讲，我一开始也犹豫过，因为在合同没签之前就先做东西，听起来多少有点冒险。

## Pattern 4: Abstract Summary -> Concrete Stakes

### Too AI-ish
客户真正关心的是效率和结果。

### Better
客户真正在意的，不是你用了什么模型，而是这件事能不能帮他少用几个人、少花多少时间，以及最后能不能顺利上线。

## Pattern 5: Fake Emotion -> Earned Emotion

### Too AI-ish
我非常激动，也非常感慨，这次经历让我收获很多。

### Better
钱打进来的那一刻，我第一个感受不是兴奋，而是终于松了一口气：原来这条路，真的能走通。

## Pattern 6: Public Account Subhead

### Too AI-ish
### 核心方法论：建立信任与提升转化

### Better
### 真正让客户开始信我的，不是报价单

## Pattern 7: Generic Closing -> Landed Closing

### Too AI-ish
如果你也想开始尝试，希望这篇文章能给你一些启发。

### Better
如果你现在也卡在那个阶段，心里知道这事也许能做，但就是不敢往前迈，那我想说，别先想着把整条路都看清。先把那个最小的版本做出来，很多信心，真的是做着做着才长出来的。

## Pattern 8: Resume Dump -> Limitation-Driven Credibility

### Too AI-ish
我之前做过医疗影像、工业视觉、机械臂监控和节能优化项目，积累了丰富经验。

### Better
本科做医疗影像时，我第一次发现，实验室里跑出来的高准确率，不等于真实场景能用。后来做工业视觉质检时，我又意识到，方案不只是要能识别，还得用得起、用得住、用得明白。

## Pattern 9: Missing Bridge -> Earned Direction Change

### Too AI-ish
后来我开始做 AI 小程序和软件创业。

### Better
也是因为前面这些经历，我后来慢慢把方向从偏工业落地，转向了 AI 小程序和轻软件创业。不是我不看好工业项目，而是它的交付周期太长了，对刚起步的人来说，先跑通现金流更重要。

## Pattern 10: Generic Personal Branding -> Trust Mechanism

### Too AI-ish
我意识到个人 IP 很重要，所以开始经营朋友圈和公众号。

### Better
我后来会把朋友圈当成一份持续更新的信任说明书。新朋友点进来，不是只看到一个服务商，而是能很快知道我在做什么、做过什么、最近在想什么。

## Pattern 11: Dense Block -> Breathable Mobile Rhythm

### Too AI-ish
我参加过航海，也意识到个人 IP 的重要性，很多技术人不擅长表达，我会发朋友圈、做公众号、做分享、参加线下活动，这些事都让我慢慢被更多人看见，也让我拿到了更多信任。

### Better
我是在参加航海之后，才越来越意识到个人 IP 的重要性。

很多技术出身的人，其实都不太擅长表达自己。

但我会把朋友圈当成一份持续更新的信任说明书。
